1. Données collectées

Nakora, développée par DOC CODE DEV, collecte uniquement les données nécessaires au fonctionnement de l'application :


  • Numéro de téléphone (identifiant principal du compte)
  • Adresse e-mail (optionnelle, pour récupération de compte)
  • Nom d'utilisateur (affiché dans votre profil)
  • Photo de profil (optionnelle)
  • Pays détecté via l'indicatif téléphonique (pour la devise et les options de paiement)
  • Token de notification push (pour les alertes de pronostics)
  • Données biométriques locales (Face ID / Touch ID) — stockées uniquement sur votre appareil, jamais transmises
  • Identifiant d'installation (pour la gestion de la période d'essai)
  • Données d'utilisation anonymisées via Firebase Analytics

2. Utilisation des données

Vos données sont utilisées exclusivement pour :


  • Authentifier votre compte et sécuriser l'accès à Nakora
  • Permettre la reconnexion rapide via biométrie (Face ID / Touch ID)
  • Détecter votre pays et devise pour l'affichage des tarifs
  • Personnaliser votre expérience (préférences, historique de notifications)
  • Vous envoyer des notifications de pronostics et résultats
  • Améliorer nos algorithmes de prédiction et l'expérience utilisateur
  • Gérer vos abonnements et paiements mobiles
  • Prévenir les abus (période d'essai, comptes multiples)

3. Stockage et sécurité

Vos données sont stockées de manière sécurisée sur des serveurs Supabase (infrastructure cloud). Les mots de passe sont hashés avec des algorithmes de chiffrement modernes. Les communications sont chiffrées via HTTPS/TLS.


Les identifiants biométriques sont stockés localement dans le stockage sécurisé de votre appareil (Keychain sur iOS, Keystore sur Android) et ne sont jamais transmis à nos serveurs.


L'historique de vos notifications est stocké localement sur votre appareil.

4. Paiements

Les paiements sont traités via PawaPay (mobile money : Orange Money, MTN MoMo, Wave) et par carte bancaire. Nakora (DOC CODE DEV) ne stocke aucune donnée financière sensible. Les transactions sont gérées intégralement par les opérateurs de paiement. Seuls le statut et la référence de la transaction sont conservés pour le suivi de votre abonnement.

5. Partage des données

Nakora (DOC CODE DEV) ne vend, ne loue et ne partage jamais vos données personnelles avec des tiers à des fins commerciales ou publicitaires.


Les seuls services tiers utilisés sont :


  • Supabase — authentification et base de données sécurisée
  • Firebase — notifications push et analytics anonymes (Google)
  • PawaPay — traitement des paiements Mobile Money

6. Vos droits

Vous disposez des droits suivants concernant vos données :


  • Accès — consultez vos données dans votre profil
  • Rectification — modifiez vos informations à tout moment
  • Suppression — Profil → Supprimer mon compte (irréversible)
  • Désactivation biométrique — désactivez Face ID / Touch ID dans Profil
  • Notifications — gérez vos préférences dans Profil → Notifications
  • Portabilité — exportez vos données sur demande écrite

7. Cookies et tracking

Nakora n'utilise pas de cookies. L'application ne contient aucun tracker publicitaire. Firebase Analytics collecte uniquement des données d'utilisation anonymes (écrans visités, événements) pour améliorer le service. Aucune donnée personnelle identifiable n'est envoyée à Firebase.

8. Modifications de cette politique

DOC CODE DEV se réserve le droit de mettre à jour cette politique de confidentialité. En cas de modification substantielle, vous serez informé via l'application Nakora. La date de dernière mise à jour est indiquée en haut de cette page.
